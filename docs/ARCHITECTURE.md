### 1.4 实时数据推送流程

```
┌──────────┐    数据更新     ┌──────────┐    事件发布    ┌──────────┐
│ 数据层   │ ───────────────>│ 接口层   │ ──────────────>│ 前端层   │
│          │                 │          │                │          │
│ 更新车辆 │                 │WebSocket │                │ 实时更新 │
│ 状态     │                 │ Handler  │                │ 界面     │
└──────────┘                 └──────────┘                └──────────┘

具体流程：
1. 数据层更新车辆状态（如位置、电量）
2. 数据层发布 vehicle_update 事件
3. 接口层的 WebSocketHandler 监听到事件
4. WebSocketHandler 将数据推送给所有订阅的前端客户端
5. 前端接收到推送数据，更新界面显示
```

### 1.5 主动查询数据流程

```
┌──────────┐    HTTP请求     ┌──────────┐    数据查询    ┌──────────┐
│ 前端层   │ ───────────────>│ 接口层   │ ──────────────>│ 数据层   │
│          │                 │          │                │          │
│ 发起查询 │                 │API       │                │ 返回数据 │
│ 请求     │                 │Controller│                │          │
└──────────┘                 └──────────┘                └──────────┘
     ↑                                                        │
     │                      HTTP响应                          │
     └────────────────────────────────────────────────────────┘

具体流程：
1. 前端发起 HTTP GET 请求（如 /api/vehicles）
2. 接口层的 APIController 接收请求
3. APIController 调用数据层的 get_vehicles() 方法
4. 数据层返回车辆数据
5. APIController 将数据转换为 JSON 格式返回给前端
6. 前端接收数据并更新界面
```

### 1.6 两种数据获取方式对比

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据获取方式对比                              │
├──────────────────────┬──────────────────────┬──────────────────┤
│       特性           │   WebSocket推送      │   REST API查询   │
├──────────────────────┼──────────────────────┼──────────────────┤
│ 数据流向             │ 服务端 → 客户端      │ 客户端 → 服务端  │
│ 实时性               │ 高（实时推送）       │ 低（需主动查询） │
│ 适用场景             │ 实时监控、状态更新   │ 历史查询、初始化 │
│ 网络开销             │ 低（长连接）         │ 高（每次请求）   │
│ 实现复杂度           │ 较高                 │ 较低             │
│ 数据一致性           │ 强（自动同步）       │ 弱（需手动刷新） │
└──────────────────────┴──────────────────────┴──────────────────┘

推荐使用场景：
- WebSocket推送：车辆位置更新、任务状态变化、充电站状态变化
- REST API查询：页面初始化、历史数据查询、配置信息获取
```

## 2. 业务逻辑图

### 2.1 静态规划业务流程

```
开始
  ↓
收集历史任务和预测任务
  ↓
数据层提供任务数据
  ↓
调度决策层接收数据
  ↓
调用算法层聚类分区算法
  ↓
将任务分组到不同区域
  ↓
对每个区域调用MIP求解器
  ↓
生成全局最优路径
  ↓
评估调度方案
  ↓
更新数据层车辆和任务状态
  ↓
存储规划结果
  ↓
结束
```

### 2.2 动态调度业务流程

```
开始
  ↓
接口层接收实时任务
  ↓
数据层存储任务信息
  ↓
调度决策层接收新任务
  ↓
策略选择器分析任务特征
  ↓
选择合适调度策略
  ↓
调用算法层实时调度策略
  ↓
生成调度指令
  ↓
评估电量是否充足
  ↓
电量充足 → 分配任务
电量不足 → 选择充电站
  ↓
更新数据层车辆状态
  ↓
返回调度结果
  ↓
结束
```

### 2.3 电量管理业务流程

```
开始
  ↓
数据层监控车辆电量
  ↓
计算到目标点的电量消耗
  ↓
判断电量是否充足
  ↓
电量充足 → 继续执行任务
电量不足 → 进入充电流程
  ↓
调用算法层路径计算
  ↓
计算到各充电站的距离
  ↓
评估充电站排队情况
  ↓
选择最优充电站
  ↓
生成充电指令
  ↓
更新数据层车辆状态
  ↓
更新数据层充电站状态
  ↓
结束
```

### 2.4 多车辆协同业务流程

```
开始
  ↓
接口层接收大重量任务
  ↓
数据层分析任务重量
  ↓
判断是否需要多车协同
  ↓
需要协同 → 进入协同流程
不需要协同 → 单车调度
  ↓
调度决策层分析可用车辆
  ↓
选择协同车辆组合
  ↓
调用算法层路径规划
  ↓
规划协同路径
  ↓
分配任务给各车辆
  ↓
更新数据层车辆状态
  ↓
返回协同调度结果
  ↓
结束
```

## 6. 中央仓库与完整路径规划

### 6.1 中央仓库概述

中央仓库是新能源物流车队协同调度系统的核心起点和终点，所有车辆任务都从仓库出发，完成任务后返回仓库。

**仓库特性**：
- 固定位置：默认位于坐标 (0, 0)，可通过 `set_warehouse_position()` 方法设置
- 车辆起点：所有空闲车辆默认位于仓库位置
- 任务终点：车辆完成任务后必须返回仓库才算完成

### 6.2 完整路径规划流程

车辆完成任务的完整路径包括三个阶段：

### 6.3 路径规划算法

#### 6.3.1 完整路径计算

```python
def calculate_complete_path(warehouse_pos, task_pos):
    """
    计算完整路径：仓库 → 任务点 → 仓库
    
    参数:
        warehouse_pos: 仓库位置 (x, y)
        task_pos: 任务点位置 (x, y)
    
    返回:
        complete_path: 完整路径点列表
        total_distance: 总距离
    """
    # 阶段1：仓库到任务点
    path_to_task = calculate_shortest_path(warehouse_pos, task_pos)
    
    # 阶段2：任务点返回仓库
    path_to_warehouse = calculate_shortest_path(task_pos, warehouse_pos)
    
    # 合并完整路径（避免重复任务点）
    complete_path = path_to_task + path_to_warehouse[1:]
    
    # 计算总距离
    total_distance = calculate_path_distance(complete_path)
    
    return complete_path, total_distance
```

#### 6.3.2 电量消耗计算（基于完整路径）

```python
def calculate_energy_consumption_for_complete_path(vehicle, complete_path):
    """
    基于完整路径计算电量消耗
    
    参数:
        vehicle: 车辆对象
        complete_path: 完整路径点列表
    
    返回:
        total_energy: 总电量消耗
        is_feasible: 电量是否充足
    """
    total_energy = 0.0
    
    # 遍历完整路径的每一段
    for i in range(len(complete_path) - 1):
        from_pos = complete_path[i]
        to_pos = complete_path[i + 1]
        
        # 计算该段距离
        distance = calculate_distance(from_pos, to_pos)
        
        # 计算该段电量消耗（考虑载重）
        energy = distance * vehicle.unit_energy_consumption
        total_energy += energy
    
    # 检查电量是否充足
    is_feasible = total_energy <= vehicle.battery
    
    return total_energy, is_feasible
```

### 6.4 任务完成判定逻辑

```python
def check_task_completion(vehicle, task):
    """
    检查任务是否完成
    
    完成条件：
    1. 车辆已到达任务点并完成配送
    2. 车辆已从任务点返回仓库
    3. 车辆状态更新为空闲
    
    参数:
        vehicle: 车辆对象
        task: 任务对象
    
    返回:
        is_completed: 任务是否完成
        completion_time: 完成时间
    """
    # 检查车辆是否在仓库位置
    if not is_at_warehouse(vehicle.position):
        return False, None
    
    # 检查任务状态
    if task.status != "in_progress":
        return False, None
    
    # 检查车辆是否分配了该任务
    if task.id not in vehicle.assigned_task_ids:
        return False, None
    
    # 检查车辆是否已完成配送（通过路径历史判断）
    if not has_completed_delivery(vehicle, task):
        return False, None
    
    # 所有条件满足，任务完成
    completion_time = current_timestamp()
    
    # 更新任务状态
    task.status = "completed"
    task.complete_time = completion_time
    
    # 更新车辆状态
    vehicle.status = "idle"
    vehicle.assigned_task_ids.remove(task.id)
    
    return True, completion_time
```

### 6.5 性能指标计算（基于完整路径）

#### 6.5.1 任务完成时间

```python
def calculate_task_completion_time(task, vehicle, complete_path):
    """
    计算任务完成时间（基于完整路径）
    
    参数:
        task: 任务对象
        vehicle: 车辆对象
        complete_path: 完整路径
    
    返回:
        completion_time: 完成时间（秒）
    """
    # 计算总距离
    total_distance = calculate_path_distance(complete_path)
    
    # 计算总时间（假设车辆速度为 1 单位/秒）
    vehicle_speed = 1.0
    total_time = total_distance / vehicle_speed
    
    # 加上任务点停留时间（假设为 30 秒）
    unloading_time = 30
    
    completion_time = total_time + unloading_time
    
    return completion_time
```

#### 6.5.2 任务得分计算

```python
def calculate_task_score(task, completion_time):
    """
    计算任务得分
    
    得分规则：
    - 按时完成：+100 分
    - 超时完成：-50 分
    - 路径长度影响：路径越短，得分越高
    
    参数:
        task: 任务对象
        completion_time: 实际完成时间
    
    返回:
        score: 任务得分
    """
    # 基础得分
    if completion_time <= task.deadline:
        base_score = 100
    else:
        base_score = -50
    
    # 路径长度影响（基于完整路径）
    # 假设平均路径长度为 1000 单位
    avg_path_length = 1000
    path_length_bonus = (avg_path_length / task.complete_path_length) * 10
    
    # 总得分
    score = base_score + path_length_bonus
    
    return score
```

### 6.6 车辆状态管理

```python
class VehicleState:
    """
    车辆状态管理
    
    状态流转：
    idle → transporting_to_task → delivering → returning_to_warehouse → idle
    """
    
    IDLE = "idle"
    TRANSPORTING_TO_TASK = "transporting_to_task"
    DELIVERING = "delivering"
    RETURNING_TO_WAREHOUSE = "returning_to_warehouse"
    CHARGING = "charging"
    
    @staticmethod
    def can_accept_task(vehicle):
        """检查车辆是否可以接受新任务"""
        return vehicle.status == VehicleState.IDLE
    
    @staticmethod
    def update_state_on_departure(vehicle, task):
        """车辆出发时的状态更新"""
        vehicle.status = VehicleState.TRANSPORTING_TO_TASK
        vehicle.assigned_task_ids.append(task.id)
        task.status = "in_progress"
        task.start_time = current_timestamp()
    
    @staticmethod
    def update_state_on_arrival(vehicle, task):
        """车辆到达任务点时的状态更新"""
        vehicle.status = VehicleState.DELIVERING
    
    @staticmethod
    def update_state_on_delivery_complete(vehicle, task):
        """配送完成，准备返回仓库"""
        vehicle.status = VehicleState.RETURNING_TO_WAREHOUSE
    
    @staticmethod
    def update_state_on_return(vehicle):
        """车辆返回仓库时的状态更新"""
        vehicle.status = VehicleState.IDLE
        vehicle.current_path = []
```

```python
        """
        task = self.tasks[task_id]
        vehicle = self.vehicles[vehicle_id]
        
        # 计算完整路径
        complete_path, total_distance = calculate_complete_path(
            self.warehouse_position, task.position
        )
        
        # 计算电量消耗
        energy_consumption, is_feasible = calculate_energy_consumption_for_complete_path(
            vehicle, complete_path
        )
        
        return complete_path, total_distance, energy_consumption, is_feasible
    
    def check_and_complete_task(self, vehicle_id):
        """
        检查并完成任务
        
        参数:
            vehicle_id: 车辆ID
        
        返回:
            completed_tasks: 已完成的任务列表
        """
        vehicle = self.vehicles[vehicle_id]
        completed_tasks = []
        
        # 检查车辆是否在仓库
        if not is_at_warehouse(vehicle.position, self.warehouse_position):
            return completed_tasks
        
        # 检查所有进行中的任务
        for task_id in vehicle.assigned_task_ids[:]:
            task = self.tasks[task_id]
            
            # 检查任务是否完成
            is_completed, completion_time = check_task_completion(vehicle, task)
            
            if is_completed:
                completed_tasks.append(task)
                
                # 计算任务得分
                task_score = calculate_task_score(task, completion_time)
                task.score = task_score
                
                # 发布任务完成事件
                self.publish_event("task_completed", task)
        
        return completed_tasks
```
